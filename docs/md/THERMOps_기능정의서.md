# 1. 솔루션 개요

본 솔루션은 열수요 예측 모델의 개발·운영 생명주기를 표준화하기 위한 오픈소스 기반 MLOps 스타터 솔루션이다. 제안 전 단계에서 공통 설치 패키지, 표준 데이터 모델, 학습/예측 파이프라인, 실험관리, 모델 버전관리, 예측 API, 대시보드를 미리 구성해두고, 수주 후 발주기관의 실제 데이터 구조와 운영환경에 맞춰 빠르게 적용하는 것을 목표로 한다.

| **솔루션 성격** | PoC 전용 코드가 아닌 사전 구축형 MLOps/예측 솔루션 골격                                                |
|-----------------|--------------------------------------------------------------------------------------------------------|
| **주요 대상**   | 지역난방 열수요 예측, 지사/권역/공급구역 단위 수요 예측                                                |
| **운영 형태**   | 정기 배치 예측 중심. 필요 시 API 조회 및 대시보드 연계                                                 |
| **핵심 가치**   | 수주 후 데이터 매핑과 모델 재학습만으로 빠른 설치·적용 가능성 확보                                     |
| **기술 방향**   | Airflow 또는 Dagster 기반 파이프라인 + MLflow 기반 실험/모델관리 + FastAPI/Grafana/Streamlit 기반 활용 |

# 2. 추진 배경 및 목표

## 2.1 추진 배경

- 열수요 예측 기능은 예측 모델 자체뿐 아니라 데이터 적재, 품질 검증, 학습 재현성, 모델 버전관리, 예측 이력관리, 성능 모니터링 체계가 함께 필요하다.

- 과제 수주 후 처음부터 MLOps 환경을 구성하면 초기 분석·설치·검증 기간이 길어질 수 있으므로, 사전에 표준화된 설치형 패키지와 기능 템플릿을 확보하는 것이 유리하다.

- 공공·에너지 분야 제안에서는 완성된 예측 성능을 단정하기보다, 데이터 기반으로 검증 가능한 운영 파이프라인과 성능 개선 체계를 제시하는 것이 안전하다.

## 2.2 목표

- 열수요 실적, 기상, 달력 데이터를 표준 스키마로 적재하고 예측 모델 학습에 활용할 수 있도록 한다.

- Airflow/Dagster 기반으로 데이터 적재, Feature 생성, 모델 학습, 배치 예측, 성능 평가, 모니터링 파이프라인을 구성한다.

- MLflow 기반으로 학습 실험, 모델 Artifact, 모델 버전, 성능지표를 관리한다.

- FastAPI와 대시보드를 통해 예측 결과, 실제값 대비 오차, 모델 성능, 파이프라인 상태를 조회할 수 있도록 한다.

- 수주 후 발주기관의 원천 데이터 구조에 맞춰 컬럼 매핑, Feature 보정, 모델 재학습, 운영 배치 조정을 신속하게 수행할 수 있게 한다.

# 4. 목표 아키텍처

## 4.1 개념 구조

| **계층**         | **구성 요소**                                                                |
|------------------|------------------------------------------------------------------------------|
| 데이터 소스      | 열수요 실적 DB/CSV, 기상 데이터, 달력/공휴일, 지사/권역 마스터               |
| 데이터 처리      | 표준 스키마 적재, 품질검증, 결측/이상치 처리, Feature 생성                   |
| MLOps 파이프라인 | Airflow/Dagster 기반 데이터 적재·학습·예측·모니터링 워크플로우               |
| 모델 관리        | MLflow 기반 실험 추적, 모델 Artifact 저장, 모델 Registry, Champion 모델 지정 |
| 저장소           | PostgreSQL 기반 메타/예측결과 저장, MinIO 기반 파일/모델 Artifact 저장       |
| 서비스/API       | FastAPI 기반 예측 결과 조회, 모델 성능 조회, 실행 이력 조회                  |
| 대시보드         | Streamlit 또는 Grafana 기반 예측/실제/오차/모델 상태 시각화                  |

## 4.2 처리 흐름

1.  원천 열수요·기상·달력 데이터를 표준 스키마로 적재한다.

2.  데이터 품질을 점검하고 결측, 중복, 시간 누락, 이상치를 검출한다.

3.  전일 수요, 전주 동일시간 수요, 이동평균, 온도 민감도, 요일/공휴일 등 Feature를 생성한다.

4.  학습·검증·테스트 기간을 분리하고 Baseline 모델과 ML 모델을 학습한다.

5.  학습 파라미터, 성능지표, 모델 파일을 MLflow에 기록하고 우수 모델을 Registry에 등록한다.

6.  등록된 Champion 모델로 D+1/D+7 배치 예측을 수행하고 예측 결과를 저장한다.

7.  실제값 수집 후 예측값과 매칭하여 MAE, RMSE, MAPE 등 성능지표를 계산한다.

8.  성능 저하 또는 데이터 분포 변화가 발생하면 재학습 후보로 표시한다.

| **영역**     | **사전 개발 산출물**                                         | **기능 목적**                                                       | **수주 후 보정 사항**                             |

# 11. 설치·배포·운영 기능

## 11.1 설치 패키지 구성

사전 구축 산출물은 Docker Compose 기반 설치를 1차 목표로 하고, 향후 운영환경 요구에 따라 Kubernetes/Helm 배포 구조로 확장한다.

| **구성 요소** | **파일/디렉터리 예시**                  | **설명**                                                        |
|---------------|-----------------------------------------|-----------------------------------------------------------------|
| 컨테이너 구성 | docker-compose.yml                      | Airflow/Dagster, MLflow, PostgreSQL, MinIO, API, Dashboard 기동 |
| 환경 변수     | .env.example                            | DB 계정, 포트, 저장소 경로, 모델명 등 설정                      |
| 파이프라인    | airflow/dags 또는 dagster/assets        | 데이터 적재, 학습, 예측, 평가 DAG/Asset                         |
| 모델 코드     | ml/features, ml/training, ml/prediction | Feature 생성, 모델 학습, 예측 로직                              |
| API           | api/                                    | FastAPI 기반 조회 API                                           |
| 대시보드      | dashboard/                              | Streamlit 또는 Grafana 설정                                     |
| DB 스키마     | db/schema.sql                           | 표준 테이블 생성 스크립트                                       |
| 문서          | docs/                                   | 설치가이드, 데이터 매핑가이드, 운영가이드                       |

## 11.2 운영 관리 기능

- 서비스 헬스체크: Airflow/Dagster, MLflow, API, DB, Object Storage 기동 상태 확인

- 파이프라인 재실행: 실패한 데이터 적재, 학습, 예측, 평가 작업의 재실행 지원

- 로그 조회: 작업 시작/종료 시각, 성공/실패 상태, 오류 메시지 관리

- 모델 롤백: 문제가 있는 모델 버전 대신 이전 Champion 모델 사용 가능하도록 Registry 기준 관리

- 설정 백업: 환경 설정, 모델 설정, 데이터 매핑 설정 파일 버전관리

# 13. 제외 범위 및 유의사항

## 13.1 사전 개발 단계 제외 범위

- 발주기관 운영계 DB, SCADA, EMS 등 실제 시스템과의 직접 연계

- 실제 데이터 기반 예측 정확도 보장 또는 특정 성능 수치 확약

- 모델 완전 자동 재학습 및 무승인 운영 반영

- 고가용성(HA), 재해복구(DR), 대규모 클러스터 운영 구성

- 상용 MLOps, BI, 데이터 통합 솔루션과의 상세 연계 구현

- 최종 통합운영 플랫폼 화면 수준의 UI/UX 완성

## 13.2 유의사항

- 사전 구축 산출물은 “완성 솔루션”이 아니라 “현장 적용 가능한 스타터 솔루션”으로 표현하는 것이 안전하다.

- 제안서에는 “오픈소스 MLOps 기반 예측 파이프라인 템플릿을 사전 확보”했다는 수준으로 기술하는 것이 적절하다.

- 실제 사업에서는 데이터 품질, 결측률, 계절별 패턴, 공급구역 특성, 기상 매핑 정확도에 따라 예측 성능이 크게 달라질 수 있다.

- 운영 중 자동 재학습은 장애와 품질 리스크가 있으므로, 1차 적용에서는 재학습 후보 알림과 승인 기반 재학습으로 제한하는 것이 바람직하다.
